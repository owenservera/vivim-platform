# VIVIM Demo Copy
## Investor Presentations · Enterprise Sales · Trade Shows · Product-Led Growth

---

## Demo Product Overview

VIVIM's demo is a **pre-seeded, cinematic walkthrough** of the complete user journey — from fragmented AI chaos to unified, context-powered intelligence. It plays like a movie or responds like an app, depending on the audience.

The demo runs on a seeded instance with 320+ conversations, 1,000+ ACUs, 150+ memories, circles, groups, and full social graph data. No live capture. No dependencies on external providers. Every search returns the right results. Every page is pre-positioned.

---

## Demo Modes

### Mode 1: Guided (Movie)
**Best for:** Trade shows, video recording, website embed, automated loops

Auto-advances through the narrative with timed transitions. Audience watches the story unfold. Branching follows the default path. Only controls: Play / Pause.

---

### Mode 2: Interactive
**Best for:** Website PLG, self-serve evaluation, product exploration

User-paced. Click or tap to advance. Branch choices are presented. No narrator — the UI tells the story.

---

### Mode 3: Presenter
**Best for:** Sales calls, investor meetings, Q&A, live demos with audience participation

Presenter has full control via a hidden keyboard-driven panel. Jump anywhere in the narrative. Trigger branching on demand. Works with real-time audience input.

---

### Mode 4: Sandbox
**Best for:** Free trial, evaluation period, developer exploration

All data loaded. No narrative. Full app navigation. This is the live product with demo data. No scaffolding.

---

### Mode 5: Kiosk
**Best for:** Trade show booths, conference kiosks, demo stations

Touch-optimized. Loop mode. Auto-reset after 60 seconds of idle. Large tap targets. Designed for 10-second attention windows with a clear restart path.

## Demo Anti-Patterns

### Never Do These

- **Live capture during demo.** Providers change. It will fail. Always use seeded data.
- **Empty states.** Pre-seed everything. The archive should be full.
- **Cold-start latency.** Keep the demo instance warm. Pre-position cameras.
- **Typing passwords.** Pre-auth. The demo should open directly to the archive.
- **Feature touring.** Tell a story, not a spec sheet. Every click should have narrative purpose.
- **Explaining the UI.** If you need to explain it, it's a UX problem. Simplify the demo.
- **Going past 90 seconds.** Before asking for questions. Stop at the close.
- **Winging any click.** Practice every click. Muscle memory. No surprises.
- **Showing settings or admin panels.** Save for follow-up. Not in the demo.
- **Generic data.** "John from San Francisco" is forgettable. "Alex Chen, Senior Engineer at a Series B startup, debugging React performance across three AI tools" is a story.
