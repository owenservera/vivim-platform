The 90-second narrative

Hook Flow Wow Land
0–15s
The pain hook
Open with 3 browser tabs — ChatGPT, Claude, Gemini — all showing different conversations. Say: "Every AI power user lives here. This is where their most valuable thinking goes to die." Close all three tabs.
15–35s
The capture magic #1
Open VIVIM. Show the archive pre-loaded with 9 providers' conversations. Click into the timeline view. Let them absorb the scale — weeks of AI work, organized. No narration needed here.
35–60s
The knowledge graph magic #2 — the wow
Switch to Canvas view. The graph animates into place — nodes by provider, edges showing semantic connections. Ask: "What if your AI conversations could remember each other?" Then type one search query — something specific and technical — and watch the graph highlight relevant nodes across providers. This is the money shot.
60–80s
Context in action magic #3
Open a new chat. Ask a question. Show the context cockpit — 8 layers loading, memories surfacing, ACUs being retrieved. The response is hyper-personalized. Say: "It knows you. Not because you told it — because it's been watching you think."
80–90s
Stop. Land it. close
"That's VIVIM. You own your AI brain. Fully." Then stop talking. The silence is intentional. Let them ask the first question.

Pre-demo checklist

Run this the night before, and again 30 minutes before the meeting.

Night before
  • ✓ Run seed reset script, verify all 250+ conversations show up
  • ✓ Verify all embeddings are pre-computed (no lag on graph load)
  • ✓ Record a full screen-capture backup at 1080p (Loom or QuickTime)
  • ✓ Test all 3 demo search queries — confirm result counts are right
  • ✓ Test canvas view — graph should render in under 3 seconds
  • ✓ Confirm demo account is logged in on the demo Chrome profile
  • ✓ Practice the script 5+ times. Time it. Should be 85–95 seconds.
Day of
  • ✓ Open only the demo Chrome profile, nothing else
  • ✓ Pre-navigate to the archive view (not homepage)
  • ✓ Disable all notifications (Slack, email, phone)
  • ! Have the Loom backup URL copied to clipboard
  • ! Have a mobile hotspot ready in case wifi fails
  • ! Run the seed reset one more time, 30 min before
  • ✗ Do NOT do a "quick live capture" during the demo — too risky

Demo killers to avoid

❌ Technical
  • ✗ Live capture during the demo — providers change, it will fail
  • ✗ Empty states anywhere — pre-seed everything
  • ✗ Cold-start latency — keep the demo instance warm
  • ✗ Typing passwords — pre-auth, always
  • ✗ Sharing your prod DB — separate demo instance
❌ Narrative
  • ✗ Feature touring — tell a story, not a spec sheet
  • ✗ Explaining the UI — if you need to, it's a UX problem
  • ✗ Going past 90 seconds before questions
  • ✗ Showing settings or admin panels — save for follow-up
  • ✗ Winging any click — every click should be practiced
The one thing that will make or break it
The knowledge graph canvas view is your weapon. Investors fund things they can feel. A zoomable, living map of someone's AI brain — with nodes glowing as search results highlight — is viscerally understandable. Make it look incredible. Pre-position the camera, pre-run the search, pre-animate the graph. That 15-second moment is worth more than everything else combined.

How the highlights system works
Each focus area is defined in demo/highlights/FOCUS_AREAS.ts and includes: narrative script (hook + body + close), demo flow (pages + wait times + notes), seed data weights, search queries to verify, and slide slugs. Running demo:highlight generates a complete package — narrative printout, click script, screenshots, MDX slides, and a printable script file.
